# 管理员权限说明（2026 版）

## 1. 权限模型

管理员判定只依赖 `users` 集合中的以下字段：

1. `isAdmin === true`
2. `permissions.canManageUsers === true`

可选兜底：环境变量 `SUPER_ADMIN_OPENIDS`（逗号分隔）。默认建议留空，仅在紧急场景使用。

## 2. 服务端校验原则

所有管理类写操作必须在云函数内做服务端鉴权，不可信任前端：

1. `setAdmin`
2. `reviewCertification`
3. `reviewCommunityWorkerCertification`
4. `removeCertification`
5. `removeCommunityWorkerCertification`
6. `deletePost`
7. `deleteComment`

## 3. 共享模块位置

管理员鉴权与校验工具位于：

1. `cloudfunctions/_shared/auth.js`
2. `cloudfunctions/_shared/validate.js`

注意：`_shared` 是共享代码目录，不是独立云函数目录。

## 4. 部署注意事项

只部署真正的云函数目录，不要部署以下目录：

1. `cloudfunctions/_shared`
2. `cloudfunctions/common`

如果误上传这些目录，可能出现 `ResourceNotFound.Function` 或“找不到函数”错误。

## 5. 管理员设置方式

通过云函数调用（示例）：

```javascript
wx.cloud.callFunction({
  name: 'setAdmin',
  data: {
    targetOpenid: '目标用户openid'
  }
})
```

## 6. 最小化暴露建议

1. 管理类函数日志中不要打印完整 openid，建议脱敏。
2. 返回前端的数据只保留必要字段，避免回传完整用户档案。
3. 所有字符串输入做长度与类型校验，拒绝超长和非字符串值。
